# SSL 底盘控制技术文档

## 1. 目标

当前 `04. code/SSL CTLBRD V0.0.2` 已实现一版可联调的底盘控制固件，覆盖两类设备：

- 底盘驱动链路：上位机串口、4 路电机控制板串口、10ms 控制调度
- 传感器链路：`MPU6050` 姿态传感器，通过 `I2C1` 读取姿态数据

本版设计原则：

- `main.c` 只保留启动和主循环
- 设备一类一模块
- 接线修改必须同步 `.ioc` 和文档

## 2. 模块结构

主入口：

- `Core/Src/main.c`
- `Core/Src/stm32f4xx_it.c`

业务模块：

- `Core/Src/ssl_chassis.c`
  - 底盘协调层
  - 控制周期调度
  - 运动学解算
  - 超时急停
- `Core/Src/ssl_host_console.c`
  - 上位机串口命令解析
  - 支持 `VEL / RAW / STOP / STATUS / HELP`
- `Core/Src/ssl_motor_board.c`
  - 电机控制板协议封装
  - 轮速帧下发
- `Core/Src/ssl_uart.c`
  - 通用串口设备初始化和发送
- `Core/Src/ssl_i2c_bus.c`
  - `I2C1` 总线初始化
  - `PB6/PB7` 硬件 I2C 读写
  - SDA 卡死恢复
- `Core/Src/ssl_mpu6050.c`
  - `MPU6050` 设备封装
  - 提供初始化、周期更新和状态读取接口
- `Core/Src/mspm0_i2c.c`
  - 对原始 `mpu6050` 例程的 I2C 兼容层
- `Core/Src/clock.c`
  - 为移植代码提供毫秒时基接口

第三方移植代码保留在：

- `Core/Src/mpu6050_vendor/mpu6050.c`
- `Core/Src/mpu6050_vendor/inv_mpu.c`
- `Core/Src/mpu6050_vendor/inv_mpu_dmp_motion_driver.c`

## 3. 控制周期

- 底盘调度周期：`10 ms`
- 通信超时急停：`500 ms`
- 状态打印周期：`100 ms`
- `MPU6050` 刷新：主循环空闲时调用 `SSL_Mpu6050_Process()`

如果 500ms 内没有新的运动命令，控制器会将 4 个轮子的目标转速清零并持续下发停机帧。

## 4. 底盘坐标系与运动学

车体坐标系定义：

- `vx`：车体前进方向线速度，单位 `m/s`
- `vy`：车体左侧方向线速度，单位 `m/s`
- `wz`：车体逆时针角速度，单位 `rad/s`

当前底盘采用“按轮子自定义角度数组解算”的方式，轮位朝向由 `ssl_chassis.c` 中的角度参数决定。

默认角度定义为：

- `front_left`：`45 deg`
- `front_right`：`-45 deg`
- `rear_left`：`135 deg`
- `rear_right`：`-135 deg`

当前工程轮位顺序是：

- `front_left`
- `front_right`
- `rear_left`
- `rear_right`

通用解算公式为：

```text
wheel_i = cos(theta_i) * vx + sin(theta_i) * vy + wz * (L + W)
```

其中：

- `theta_i`：第 `i` 个轮子的自定义角度，单位 `deg`
- 角度参考方向：从车体前进方向 `+vx` 起，逆时针为正
- `L + W`：当前代码里使用 `kHalfWheelbaseM + kHalfTrackM`

默认参数位于 `ssl_chassis.c`：

- `kWheelRadiusM = 0.05f`
- `kHalfWheelbaseM = 0.12f`
- `kHalfTrackM = 0.11f`
- `kMaxWheelRpm = 450.0f`
- `kWheelAngleDeg[4] = {45, -45, 135, -135}`

如果后续机械布局变化，只需要改：

```c
static const float kWheelAngleDeg[SSL_MOTOR_BOARD_COUNT] = {45.0f, -45.0f, 135.0f, -135.0f};
```

如果电机实际正转方向和当前定义相反，改：

```c
static const int8_t kWheelDirectionSign[SSL_MOTOR_BOARD_COUNT] = {1, 1, 1, 1};
```

把对应轮位改成 `-1` 即可。

## 5. 上位机命令接口

上位机通过 `USART1` 发送 ASCII 命令，行结束支持 `\r\n` 或 `\n`。

速度命令：

```text
VEL <vx_mps> <vy_mps> <wz_radps>
```

轮速直通：

```text
RAW <fl_rpm> <fr_rpm> <rl_rpm> <rr_rpm>
```

停机命令：

```text
STOP
```

状态查询：

```text
STATUS
```

帮助命令：

```text
HELP
```

## 6. 电机控制板协议

当前版本使用 6 字节下发帧：

```text
Byte0  0x7B
Byte1  node_id
Byte2  rpm_low
Byte3  rpm_high
Byte4  checksum
Byte5  0x7D
```

说明：

- `rpm` 使用 `int16_t`
- `node_id` 取值 `1~4`
- `checksum = Byte1 + Byte2 + Byte3` 的低 8 位

如果后续 `carmove_usart` 的实际协议不同，只需要改 `ssl_motor_board.c` 的组帧逻辑，不需要改 `main.c`。

## 7. MPU6050 设备说明

当前移植使用硬件 `I2C1`，接入引脚如下：

- `PB6` -> `I2C1_SCL`
- `PB7` -> `I2C1_SDA`

软件分层：

- `ssl_i2c_bus`：STM32F4 HAL I2C 总线层
- `mspm0_i2c`：对原始例程的兼容适配层
- `ssl_mpu6050`：SSL 工程内的设备层
- `mpu6050_vendor`：移植的第三方算法与驱动源文件

`ssl_mpu6050` 对外提供：

- `SSL_Mpu6050_Init()`
- `SSL_Mpu6050_Process()`
- `SSL_Mpu6050_GetState()`

状态结构包含：

- 三轴陀螺仪 `gyro[3]`
- 三轴加速度 `accel[3]`
- 欧拉角 `pitch / roll / yaw`

## 8. 反馈与预留接口

当前保留 `EXTI0~3` 作为轮速反馈输入预留口：

- `EXTI0` -> 轮 1
- `EXTI1` -> 轮 2
- `EXTI2` -> 轮 3
- `EXTI3` -> 轮 4

这部分目前只做计数入口预留，尚未绑定具体测速硬件。后续新增反馈设备时，继续保持设备模块化，不把逻辑回灌到 `main.c`。

## 9. 接线变更同步规则

凡是新增或修改设备接线，必须同步修改以下三处：

1. 对应设备模块源码
2. `SSL CTLBRD V0.0.2.ioc`
3. `07. techdocs/02. 接线文档.md`

## 10. 默认下载方式

当前工程默认使用 `J-Link` 下载和调试。

约定如下：

- 调试接口：`SWD`
- 目标器件：`STM32F407VE`
- 默认速度：`5000 kHz`
- 工程侧配置文件：
  - `MDK-ARM/SSL CTLBRD V0.0.2.uvoptx`
  - `MDK-ARM/JLinkSettings.ini`
