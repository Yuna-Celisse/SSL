# SSL 底盘控制技术文档

## 1. 目标

本版本在 `04. code/SSL-CTLBRD V0.0.1` 中完成了一个可联调的底盘控制基础固件，解决三件事：

1. 控制板通过 `USART1` 接收上位机速度命令。
2. 控制板按 10ms 周期做四轮全向底盘运动学解算。
3. 控制板通过 4 路串口向四块电机控制板下发目标转速。

当前实现适合作为整车联调起点，后续如果你们的电机板协议或 PCB 引脚和本文档不同，只需要改 `main.c` 里的串口映射和发送帧格式。

## 2. 软件结构

主工程入口文件：

- `04. code/SSL-CTLBRD V0.0.1/Core/Src/main.c`
- `04. code/SSL-CTLBRD V0.0.1/Core/Src/stm32f4xx_it.c`

实现内容：

- `SSL_ConfigureHostInterface()`
  - 初始化 `USART1`
  - 作为上位机调试口和命令口
- `SSL_ConfigureMotorInterfaces()`
  - 初始化 `USART2/USART3/UART4/UART5`
  - 分别连接四个电机控制板
- `SSL_ConfigureTimer6()`
  - `TIM6` 10ms 周期中断
  - 作为底盘控制调度心跳
- `SSL_ChassisControlTick()`
  - 通信超时检查
  - 运动学解算
  - 下发四轮目标转速
  - 周期状态输出
- `SSL_ProcessHostCommand()`
  - 处理 `VEL / RAW / STOP / STATUS / HELP`

## 3. 控制周期

- 控制周期：`10 ms`
- 超时急停：`500 ms`
- 状态打印周期：`100 ms`

如果 500ms 内没有收到新的控制命令，控制板会将四个轮子的目标值全部清零并持续发送 0 转速帧。

## 4. 坐标系定义

采用常见车体坐标系：

- `vx`：车体前进方向线速度，单位 `m/s`
- `vy`：车体左侧方向线速度，单位 `m/s`
- `wz`：车体逆时针角速度，单位 `rad/s`

## 5. 运动学模型

程序默认按四轮全向/麦克纳姆底盘方式解算，参数在 `main.c` 中定义：

- 轮半径 `kWheelRadiusM = 0.05 m`
- 半轴距 `kHalfWheelbaseM = 0.12 m`
- 半轮距 `kHalfTrackM = 0.11 m`

解算公式：

```text
front_left  = vx - vy - wz * (L + W)
front_right = vx + vy + wz * (L + W)
rear_left   = vx + vy - wz * (L + W)
rear_right  = vx - vy + wz * (L + W)
```

再将轮面线速度换算为转速 `rpm`，并做限幅：

- 最大转速：`±450 rpm`

如果你们实际轮子安装方向与当前假设不同，只需要改：

```c
static const int8_t kWheelDirectionSign[4] = {1, 1, 1, 1};
```

把对应轮子的符号改成 `-1` 即可。

## 6. 上位机命令接口

上位机通过 `USART1` 发送 ASCII 文本命令，行结束符为 `\r\n` 或 `\n`。

### 6.1 速度控制

```text
VEL <vx_mps> <vy_mps> <wz_radps>
```

示例：

```text
VEL 0.50 0.00 0.00
VEL 0.00 0.30 0.00
VEL 0.00 0.00 1.20
```

### 6.2 直接下发四轮转速

```text
RAW <fl_rpm> <fr_rpm> <rl_rpm> <rr_rpm>
```

示例：

```text
RAW 120 120 120 120
RAW 150 -150 150 -150
```

`RAW` 模式用于绕过运动学层，直接联调四个轮子。

### 6.3 急停

```text
STOP
```

### 6.4 状态查询

```text
STATUS
```

### 6.5 帮助

```text
HELP
```

## 7. 控制板到电机板的串口帧

当前版本定义了一个简洁的 6 字节下发帧：

```text
Byte0  0x7B            帧头
Byte1  node_id         电机编号，1~4
Byte2  rpm_low         目标转速低字节
Byte3  rpm_high        目标转速高字节
Byte4  checksum        Byte1+Byte2+Byte3 的 8bit 累加和
Byte5  0x7D            帧尾
```

转速使用 `int16_t`，单位是 `rpm`。

电机编号定义：

- `1` 前左轮
- `2` 前右轮
- `3` 后左轮
- `4` 后右轮

如果 `carmove_usart` 电机板最终采用别的协议，只要保留 `SSL_SendMotorFrame()` 这个函数入口，替换里面的组帧逻辑即可。

## 8. 反馈接口

工程里保留了 `EXTI0~3` 中断入口：

- `EXTI0` -> 轮 1
- `EXTI1` -> 轮 2
- `EXTI2` -> 轮 3
- `EXTI3` -> 轮 4

当前默认仅做脉冲计数入口预留，不强依赖。后续如果你们要接霍尔脉冲、编码器 Z 相或测速中断，可以在：

- `SSL_ConfigurePulseInputs()`
- `SSL_EXTI_IRQHandler()`

里继续补完整采样逻辑。

## 9. 联调建议

建议按下面顺序联调：

1. 先只接 `USART1`，确认串口能收到 `SSL chassis controller ready.`
2. 发送 `HELP`，确认命令解析正常。
3. 在不接电机板时发送 `RAW 100 100 100 100`，用示波器/逻辑分析仪看 4 路串口是否稳定输出。
4. 单独接 1 块电机板，确认 `node_id` 和转速方向。
5. 逐个接入 4 块电机板。
6. 最后再用 `VEL` 命令验证整车平移和自转方向是否一致。
