# SSL 底盘控制技术文档

## 1. 项目范围

当前控制板工程为：

- `04. code/SSL CTLBRD V0.0.2`

当前并行项目为：

- `04. code/SSL ESP32 Bridge V0.0.1`
- `04. code/SSL Host WiFi Console V0.0.1`

本版本完成了三部分联动：

- STM32F407VE 底盘控制固件
- ESP32-C5-WROOM-1U Wi-Fi 串口桥接固件
- 上位机 Wi-Fi 操控程序

设计原则：

- `main.c` 只保留启动和主循环
- 设备一类一模块
- 接线修改必须同步到 `.ioc` 和接线文档

## 3. 控制周期

- 底盘控制周期：`10 ms`
- 通讯超时急停：`500 ms`
- 周期状态回传：`100 ms`
- `MPU6050` 更新：主循环空闲时调用 `SSL_Mpu6050_Process()`

如果在 `500 ms` 内没有收到新的速度命令、原始转速命令或停止命令，控制器会强制急停。

## 4. 底盘运动学

车体坐标定义：

- `vx`：车体前进方向线速度，单位 `m/s`
- `vy`：车体左侧方向线速度，单位 `m/s`
- `wz`：车体逆时针角速度，单位 `rad/s`

当前工程采用“可自定义轮子角度数组”的通用解算方式。

默认轮位顺序：

1. `front_left`
2. `front_right`
3. `rear_left`
4. `rear_right`

默认角度数组：

```c
static const float kWheelAngleDeg[SSL_MOTOR_BOARD_COUNT] = {45.0f, -45.0f, 135.0f, -135.0f};
```

通用逆运动学公式：

```text
wheel_i = cos(theta_i) * vx + sin(theta_i) * vy + wz * (L + W)
```

其中：

- `theta_i`：第 `i` 个轮子的驱动方向角，单位 `deg`
- 角度参考方向：从车体前进方向 `+vx` 起，逆时针为正
- `L + W`：当前代码使用 `kHalfWheelbaseM + kHalfTrackM`

默认机械参数：

- `kWheelRadiusM = 0.05f`
- `kHalfWheelbaseM = 0.12f`
- `kHalfTrackM = 0.11f`
- `kMaxWheelRpm = 450.0f`

如果轮子机械布局变化，只修改 `kWheelAngleDeg[]`。如果某个电机方向与定义相反，只修改：

```c
static const int8_t kWheelDirectionSign[SSL_MOTOR_BOARD_COUNT] = {1, 1, 1, 1};
```

## 5. 本地串口控制台协议

本地控制台使用 `USART1`，波特率 `115200`，文本协议为 ASCII。

支持命令：

```text
VEL <vx_mps> <vy_mps> <wz_radps>
RAW <fl_rpm> <fr_rpm> <rl_rpm> <rr_rpm>
STOP
STATUS
HELP
```

## 6. ESP32 Wi-Fi 通讯架构

本项目使用 `ESP32-C5-WROOM-1U` 作为 Wi-Fi 到 UART 的透明桥接模块。

链路分层如下：

- 上位机 `TCP Client`
- ESP32 `TCP Server`
- ESP32 `UART`
- STM32 `USART6`
- 底盘控制模块

本版本没有把应用层协议放在 ESP32 里解析，而是采用“透明转发”：

- ESP32 只负责网络连接和字节流转发
- STM32 与上位机共享同一套二进制帧协议

这样做的好处是：

- ESP32 固件保持简单
- 协议变更只需维护 STM32 与上位机两端
- 联调时容易抓包定位

## 7. ESP32 二进制协议

协议文件：

- STM32：`Core/Inc/ssl_esp32_protocol.h`
- STM32：`Core/Src/ssl_esp32_protocol.c`
- 上位机：`SSL Host WiFi Console V0.0.1/protocol.py`

帧格式：

```text
Byte0   0x53
Byte1   0x4C
Byte2   version
Byte3   type
Byte4   payload_length
Byte5~  payload
Last    crc8
```

说明：

- 帧头固定为 `0x53 0x4C`
- 当前版本号为 `0x01`
- `crc8` 计算范围为 `version + type + length + payload`
- CRC 多项式为 `0x07`

消息类型：

- `0x01` `VEL`
  - 负载：`<float vx, float vy, float wz>`
- `0x02` `RAW`
  - 负载：`<int16 fl, int16 fr, int16 rl, int16 rr>`
- `0x03` `STOP`
  - 负载长度：`0`
- `0x04` `STATUS_REQ`
  - 负载长度：`0`
- `0x05` `PING`
  - 负载长度：`0`
- `0x81` `ACK`
  - 负载长度：`0`
- `0x82` `STATUS`
  - 负载：`<float vx, float vy, float wz, int16 fl, int16 fr, int16 rl, int16 rr>`
- `0x83` `ERROR`
  - 负载：`<uint8 error_code>`

当前 STM32 行为：

- 收到 `VEL / RAW / STOP` 后执行命令并回 `ACK`
- 收到 `STATUS_REQ` 后回 `STATUS`
- 收到 `PING` 后回 `ACK`
- 收到非法帧或无法识别的帧后回 `ERROR`

## 8. ESP32 桥接固件

工程路径：

- `04. code/SSL ESP32 Bridge V0.0.1`

默认参数：

- Wi-Fi 模式：`STA`
- TCP 监听端口：`9000`
- UART 波特率：`921600`
- UART RX 引脚：`GPIO12`
- UART TX 引脚：`GPIO11`

默认行为：

- 连接已有 Wi-Fi 路由器
- 监听单客户端 TCP 连接
- 把 TCP 收到的数据发到串口
- 把串口收到的数据发回 TCP

## 9. 上位机程序

工程路径：

- `04. code/SSL Host WiFi Console V0.0.1`

运行环境：

- `Python 3.10+`
- 仅使用标准库

当前功能：

- 连接 ESP32 TCP 服务器
- 发送 `VEL`
- 发送 `RAW`
- 发送 `STOP`
- 发送 `STATUS`
- 发送 `PING`
- 显示 `ACK / ERROR / STATUS`

## 10. `.ioc` 同步要求

只要接线发生变化，必须同步修改以下三处：

1. 对应设备模块源码
2. `04. code/SSL CTLBRD V0.0.2/SSL CTLBRD V0.0.2.ioc`
3. `07. techdocs/02. 接线文档.md`

本次 ESP32 接入已同步到 `.ioc` 的资源：

- `USART6`
- `PC6 = USART6_TX`
- `PC7 = USART6_RX`
- `PB0 = ESP32_EN`
- `PB1 = ESP32_BOOT`
