# SSL 底盘接线文档

## 1. 说明

本接线文档对应当前仓库里的控制板固件实现，基于 `STM32F407ZET6` 控制板工程：

- `04. code/SSL-CTLBRD V0.0.1`

由于 `03. PCB` 目录当前没有可直接读取的原理图文件，本文件先按固件默认引脚分配给出接线方案。若你们实物 PCB 已经定板，只要把 `main.c` 中对应串口引脚改成实物版本即可。

## 2. 接口分配总表

| 功能 | MCU 外设 | TX 引脚 | RX 引脚 | 对端 |
| --- | --- | --- | --- | --- |
| 上位机调试/控制口 | `USART1` | `PA9` | `PA10` | 上位机串口/USB 转串口 |
| 前左电机 | `USART2` | `PD5` | `PD6` | 电机控制板 1 |
| 前右电机 | `USART3` | `PD8` | `PD9` | 电机控制板 2 |
| 后左电机 | `UART4` | `PC10` | `PC11` | 电机控制板 3 |
| 后右电机 | `UART5` | `PC12` | `PD2` | 电机控制板 4 |

## 3. 上位机接线

### 3.1 USB 转串口连接

| USB 转串口 | 控制板 |
| --- | --- |
| TXD | `PA10` |
| RXD | `PA9` |
| GND | GND |

注意：

1. 串口必须交叉连接，`TX -> RX`、`RX -> TX`
2. 地必须共地
3. 电平必须是 `3.3V TTL`
4. 默认波特率 `115200`

## 4. 四个电机控制板接线

### 4.1 串口接线规则

每块电机控制板都按同样规则接：

| 控制板 | 电机板 |
| --- | --- |
| TX | RX |
| RX | TX |
| GND | GND |

### 4.2 分轮接线

#### 前左轮

| 控制板 | 电机板 |
| --- | --- |
| `PD5` (`USART2_TX`) | 电机板 RX |
| `PD6` (`USART2_RX`) | 电机板 TX |
| GND | 电机板 GND |

#### 前右轮

| 控制板 | 电机板 |
| --- | --- |
| `PD8` (`USART3_TX`) | 电机板 RX |
| `PD9` (`USART3_RX`) | 电机板 TX |
| GND | 电机板 GND |

#### 后左轮

| 控制板 | 电机板 |
| --- | --- |
| `PC10` (`UART4_TX`) | 电机板 RX |
| `PC11` (`UART4_RX`) | 电机板 TX |
| GND | 电机板 GND |

#### 后右轮

| 控制板 | 电机板 |
| --- | --- |
| `PC12` (`UART5_TX`) | 电机板 RX |
| `PD2` (`UART5_RX`) | 电机板 TX |
| GND | 电机板 GND |

## 5. 供电要求

### 5.1 控制板

- 控制板供电按你们控制板设计输入
- MCU 侧串口 IO 电平为 `3.3V`

### 5.2 电机板

- 电机动力电源按 BM50 电机板要求供电
- 控制板和 4 块电机板必须共地

如果不共地，串口大概率会出现误码或完全无法通信。

## 6. 轮位定义

固件里的轮位顺序固定为：

1. `front_left`
2. `front_right`
3. `rear_left`
4. `rear_right`

接线时建议直接在每根线束和每块电机板上贴标签：

- `M1-FL`
- `M2-FR`
- `M3-RL`
- `M4-RR`

这样后续查方向和协议更快。

## 7. 可选反馈接线

固件保留了 4 路外部中断入口，可作为测速脉冲输入使用：

| 反馈通道 | EXTI 线 |
| --- | --- |
| 轮 1 | `EXTI0` |
| 轮 2 | `EXTI1` |
| 轮 3 | `EXTI2` |
| 轮 4 | `EXTI3` |

当前代码只保留了入口，默认没有绑定具体 GPIO 口，也没有在文档里强制指定线序。原因是你们当前仓库缺少最终原理图，实际应该按 PCB 网络名再定。

## 8. 上电前检查清单

1. 所有串口都是交叉连接
2. 所有设备共地
3. 没有把 `5V TTL` 直接打到 `3.3V` MCU IO
4. 四个轮子的编号和固件定义一致
5. 电机机械安装方向和轮位标签一致
6. 首次调试先空载悬空，不要直接落地跑

## 9. 首次联调步骤

1. 先只接上位机串口，不接 4 个电机板
2. 打开串口助手，`115200 8N1`
3. 复位控制板，观察启动信息
4. 发送 `HELP`
5. 再逐个接入电机板
6. 用 `RAW 100 0 0 0`、`RAW 0 100 0 0` 逐个点轮验证
7. 最后用 `VEL 0.2 0 0` 做整车平移验证

## 10. 如果实际 PCB 不同，改哪里

只需要改 `main.c` 里这两部分：

1. `kHostPort`
2. `g_motors[]`

改完后重新编译下载即可，其他上层底盘逻辑不用动。
