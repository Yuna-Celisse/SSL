# SSL 接线文档

## 1. 说明

本文档对应当前仓库中的控制板工程：

- `04. code/SSL CTLBRD V0.0.2`

当前接线定义同时对应以下三处内容：

1. 设备模块源码
2. `SSL CTLBRD V0.0.2.ioc`
3. 本文档

后续只要接线发生变化，三处必须同步。

## 2. 接口总表

| 功能 | MCU 外设 | 引脚 | 对端 |
| --- | --- | --- | --- |
| 上位机调试/控制串口 | `USART1` | `PA9/PA10` | USB 转串口 |
| 前左电机控制板 | `USART2` | `PD5/PD6` | 电机板 1 |
| 前右电机控制板 | `USART3` | `PD8/PD9` | 电机板 2 |
| 后左电机控制板 | `UART4` | `PC10/PC11` | 电机板 3 |
| 后右电机控制板 | `UART5` | `PC12/PD2` | 电机板 4 |
| `MPU6050` 姿态传感器 | `I2C1` | `PB6/PB7` | `MPU6050` |

## 3. 上位机串口接线

| USB 转串口 | 控制板 |
| --- | --- |
| TXD | `PA10` |
| RXD | `PA9` |
| GND | GND |

要求：

1. 串口必须交叉连接，`TX -> RX`、`RX -> TX`
2. 地必须共地
3. 电平必须是 `3.3V TTL`
4. 默认波特率 `115200`

## 4. 四路电机控制板接线

通用规则：

| 控制板 | 电机板 |
| --- | --- |
| TX | RX |
| RX | TX |
| GND | GND |

轮位分配：

1. `front_left`
2. `front_right`
3. `rear_left`
4. `rear_right`

当前四轮全向轮机械布局角度：

- `front_left`：`45 deg`
- `rear_left`：`135 deg`
- `rear_right`：`-135 deg`
- `front_right`：`-45 deg`

建议在线束和电机板上直接标记：

- `M1-FL`
- `M2-FR`
- `M3-RL`
- `M4-RR`

具体引脚：

| 轮位 | 控制板引脚 | 串口功能 |
| --- | --- | --- |
| 前左 | `PD5` / `PD6` | `USART2_TX` / `USART2_RX` |
| 前右 | `PD8` / `PD9` | `USART3_TX` / `USART3_RX` |
| 后左 | `PC10` / `PC11` | `UART4_TX` / `UART4_RX` |
| 后右 | `PC12` / `PD2` | `UART5_TX` / `UART5_RX` |

## 5. MPU6050 接线

当前 SSL 中的 `MPU6050` 使用硬件 I2C 接入。

| MPU6050 引脚 | 控制板引脚 | 说明 |
| --- | --- | --- |
| `VCC` | `3.3V` | 传感器供电 |
| `GND` | `GND` | 共地 |
| `SCL` | `PB6` | `I2C1_SCL` |
| `SDA` | `PB7` | `I2C1_SDA` |
| `AD0` | `GND` | 默认地址 `0x68` |
| `INT` | 暂未使用 | 当前固件未接入中断脚 |

要求：

1. 传感器侧总线电平保持 `3.3V`
2. `SCL`、`SDA` 需要上拉
3. 传感器模块如果板上已经自带上拉，外部不要重复堆太低阻值
4. 当前固件已实现 SDA 卡死恢复，但不替代正确硬件接线

## 6. 轮速反馈预留口

当前保留了 4 路外部中断测速入口：

| 反馈通道 | EXTI |
| --- | --- |
| 轮 1 | `EXTI0` |
| 轮 2 | `EXTI1` |
| 轮 3 | `EXTI2` |
| 轮 4 | `EXTI3` |

当前代码只预留入口，尚未绑定具体 GPIO。等 PCB 最终定版后，再把实际测速器件接线同步到 `.ioc` 和文档。

## 7. 供电要求

- 控制板 MCU 侧 IO 为 `3.3V`
- 4 块电机板必须与控制板共地
- `MPU6050` 必须与控制板共地
- 不要把 `5V TTL` 直接接到 STM32 串口或 I2C 引脚

## 8. 上电前检查

1. 所有串口是否交叉连接
2. 所有设备是否共地
3. `MPU6050` 的 `VCC` 是否为 `3.3V`
4. `I2C1` 的 `SCL/SDA` 是否接反
5. 4 个轮位是否与固件定义一致
6. 首次联调时先空载悬空测试

## 9. 联调顺序建议

1. 先只接上位机串口，确认串口可收到启动信息
2. 发送 `HELP`，确认命令解析正常
3. 不接电机板时验证 4 路串口是否有下发数据
4. 逐个接入电机板，验证轮位和转向
5. 接入 `MPU6050`，确认 `PB6/PB7` 波形正常
6. 再做整车 `VEL` 命令联调

## 10. 如果实际 PCB 不同，改哪里

不要再直接改 `main.c`。

应按设备模块分别修改：

1. 上位机串口：`ssl_chassis.c` 中的 `kHostPort`
2. 电机板串口：`ssl_chassis.c` 中的 `g_motors[]`
3. `MPU6050` I2C：`ssl_i2c_bus.c`
4. 同步更新：`SSL CTLBRD V0.0.2.ioc`
5. 同步更新：本接线文档

## 11. 默认烧录方式

当前工程默认使用 `J-Link` 通过 `SWD` 烧录。

J-Link 接线：

| J-Link | 控制板 |
| --- | --- |
| `VTref` | `3.3V` |
| `SWDIO` | `PA13` |
| `SWCLK` | `PA14` |
| `GND` | `GND` |
| `RESET` | `NRST` |

说明：

1. `VTref` 只用于电平参考，不代替主供电
2. 默认目标器件为 `STM32F407VE`
3. 默认接口为 `SWD`
