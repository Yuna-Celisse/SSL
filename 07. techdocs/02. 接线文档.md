# SSL 接线文档

## 1. 适用工程

当前文档对应：

- `04. code/SSL CTLBRD V0.0.2`
- `04. code/SSL ESP32 Bridge V0.0.1`

本文档内容与以下两处保持同步：

1. 控制板设备模块源码
2. `SSL CTLBRD V0.0.2.ioc`

## 2. 控制板接口总表

| 功能 | MCU 外设 | 引脚 | 对端 |
| --- | --- | --- | --- |
| 本地上位机串口 | `USART1` | `PA9/PA10` | USB 转串口 |
| 前左电机控制板 | `USART2` | `PD5/PD6` | 电机板 1 |
| 前右电机控制板 | `USART3` | `PD8/PD9` | 电机板 2 |
| 后左电机控制板 | `UART4` | `PC10/PC11` | 电机板 3 |
| 后右电机控制板 | `UART5` | `PC12/PD2` | 电机板 4 |
| `MPU6050` | `I2C1` | `PB6/PB7` | 姿态传感器 |
| `ESP32-C5-WROOM-1U` 数据串口 | `USART6` | `PC6/PC7` | `GPIO12/GPIO11` |
| `ESP32-C5-WROOM-1U` 使能 | `GPIO` | `PB0` | `EN` |
| `ESP32-C5-WROOM-1U` 启动控制 | `GPIO` | `PB1` | `GPIO28` |

## 3. 本地串口接线

| USB 转串口 | 控制板 |
| --- | --- |
| `TXD` | `PA10` |
| `RXD` | `PA9` |
| `GND` | `GND` |

要求：

1. 串口必须交叉连接
2. 地必须共地
3. 电平必须为 `3.3V TTL`
4. 默认波特率为 `115200`

## 4. 四路电机控制板接线

通用规则：

| 控制板 | 电机板 |
| --- | --- |
| `TX` | `RX` |
| `RX` | `TX` |
| `GND` | `GND` |

轮位定义：

1. `front_left`
2. `front_right`
3. `rear_left`
4. `rear_right`

当前默认角度布局：

- `front_left = 45 deg`
- `front_right = -45 deg`
- `rear_left = 135 deg`
- `rear_right = -135 deg`

具体串口分配：

| 轮位 | 控制板引脚 | 串口功能 |
| --- | --- | --- |
| 前左 | `PD5 / PD6` | `USART2_TX / USART2_RX` |
| 前右 | `PD8 / PD9` | `USART3_TX / USART3_RX` |
| 后左 | `PC10 / PC11` | `UART4_TX / UART4_RX` |
| 后右 | `PC12 / PD2` | `UART5_TX / UART5_RX` |

## 5. MPU6050 接线

| MPU6050 引脚 | 控制板引脚 | 说明 |
| --- | --- | --- |
| `VCC` | `3.3V` | 供电 |
| `GND` | `GND` | 共地 |
| `SCL` | `PB6` | `I2C1_SCL` |
| `SDA` | `PB7` | `I2C1_SDA` |
| `AD0` | `GND` | 默认地址 `0x68` |
| `INT` | 暂未使用 | 当前固件未接入 |

要求：

1. `I2C1` 总线电平保持 `3.3V`
2. `SCL / SDA` 需要上拉
3. 如果传感器模块已自带上拉，不要重复并过低阻值

## 6. ESP32-C5-WROOM-1U 接线

默认控制方式为：STM32 通过 `USART6 + 2 路 GPIO` 与 ESP32 模块连接，ESP32 再通过 Wi-Fi 与上位机通信。

### 6.1 STM32 与 ESP32 对接表

| STM32 控制板 | ESP32-C5-WROOM-1U | 说明 |
| --- | --- | --- |
| `PC6 / USART6_TX` | `GPIO12 / U0RXD` | STM32 发到 ESP32 |
| `PC7 / USART6_RX` | `GPIO11 / U0TXD` | ESP32 发到 STM32 |
| `PB0` | `EN` | 模块使能 / 复位控制 |
| `PB1` | `GPIO28` | 启动模式控制 |
| `3.3V` | `3V3` | 模块供电 |
| `GND` | `GND` | 共地 |

### 6.2 上电要求

1. ESP32 模块供电必须为 `3.3V`
2. 模块与 STM32 必须共地
3. `EN` 不能悬空
4. 正常启动时保持 `GPIO28` 为高
5. 若需要进入下载模式，则在复位时拉低 `GPIO28`

### 6.3 当前固件默认参数

- `USART6` 波特率：`921600`
- Wi-Fi 模式：`STA`
- TCP 端口：`9000`

## 7. 预留测速反馈口

当前保留 4 路外部中断测速输入：

| 反馈通道 | EXTI |
| --- | --- |
| 轮 1 | `EXTI0` |
| 轮 2 | `EXTI1` |
| 轮 3 | `EXTI2` |
| 轮 4 | `EXTI3` |

当前代码只保留逻辑入口，尚未绑定最终测速硬件引脚。PCB 最终定版后，需要同步修改 `.ioc` 和本文档。

## 8. 供电要求

- 控制板 MCU IO 电平：`3.3V`
- ESP32 模块供电：`3.3V`
- `MPU6050` 供电：`3.3V`
- 电机控制板、ESP32、MPU6050 必须与控制板共地
- 不允许把 `5V TTL` 直接接入 STM32 或 ESP32 的 IO

## 9. 联调顺序建议

1. 先接 `J-Link`，确认控制板可正常下载
2. 只接本地调试串口，确认 `USART1` 启动信息正常
3. 接入 ESP32，确认 `USART6` 收发正常
4. 烧录 `SSL ESP32 Bridge V0.0.1`，确认可连接目标 Wi-Fi
5. 运行 `SSL Host WiFi Console V0.0.1`，确认 `PING` 与 `STATUS` 正常
6. 再接入四路电机控制板
7. 最后做整车 `VEL / RAW / STOP` 联调

## 10. 如果实际 PCB 不同，修改位置

不要把设备逻辑改回 `main.c`。应按模块修改：

1. 本地串口：`ssl_chassis.c` 中的 `kHostPort`
2. ESP32 串口与控制脚：`ssl_chassis.c` 中的 `kEsp32LinkConfig`
3. 四路电机串口：`ssl_chassis.c` 中的 `g_motors[]`
4. `MPU6050` I2C：`ssl_i2c_bus.c`
5. 同步更新：`SSL CTLBRD V0.0.2.ioc`
6. 同步更新：本文档

## 11. 默认烧录方式

当前控制板默认使用 `J-Link` 通过 `SWD` 烧录：

| J-Link | 控制板 |
| --- | --- |
| `VTref` | `3.3V` |
| `SWDIO` | `PA13` |
| `SWCLK` | `PA14` |
| `GND` | `GND` |
| `RESET` | `NRST` |
